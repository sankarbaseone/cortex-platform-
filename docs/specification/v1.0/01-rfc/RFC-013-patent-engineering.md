# RFC-013 — Patent Engineering

**Status:** Approved · **Extends:** V1 Phase 16 (all 22 items) · **Owns Section R.**
Format per patent: Architecture · Algorithm · Claims (independent sketch) · Implementation ref · Prior art to distinguish · Commercial value · Future extensions. Legal counsel drafts final claims; this is the engineering disclosure. Trade-secret carve-outs (NOT to be filed, per V1 Phase 12): graph schema details, residual-model weights, SASS heuristic tables.

**Filing priority tiers:** Tier-1 (file pre-GA): #1, #2, #3, #5, #7. Tier-2 (Y2): #4, #6, #8, #9, #16, #18. Tier-3: remainder, file on productization.

1. **Kernel Efficiency Score** — Arch: RFC-002 §2.5 pipeline. Algo: weighted geometric composite with workload-profile weight selection and per-arch calibration. Claim sketch: method comprising canonicalizing compiler IR, computing multi-factor sub-scores from static features and hardware counters, combining via calibrated multiplicative model, emitting explainable score with versioned model id. Prior art to distinguish: MFU (workload-level, not per-kernel, not explainable), NCU speed-of-light (per-metric, no composite/calibration/identity). Impl: `analyzers/kes`. Value: category-standard metric. Extensions: cross-vendor normalization (#22 merges here as dependent claims).
2. **Compiler Regression Index & CI gate** — Algo: RFC-002 §2.6 static pre-screen classifier + targeted benchmarking + fleet time-share aggregation + threshold gate in CI. Distinguish: generic perf-regression CI (no compiler-artifact features, no fleet weighting), Yoshida-2024 (analysis, no productized detection/gating). Claims: detecting performance regressions attributable to toolchain change via IR/SASS differential features triggering selective benchmark execution and gating deployment. Value: unowned failure mode.
3. **Privacy-preserving cross-tenant kernel knowledge graph** — Arch: RFC-003 D.9 (hash identity, feature allowlist, salted contributor ids, k-anonymity gating). Claims: aggregating compiler-artifact performance facts across tenants using content-derived identifiers and enforced feature allowlists such that recommendations improve globally without raw-artifact disclosure. Distinguish: federated learning (model-gradient based), generic telemetry pooling (no content-addressed compiler identity). Value: THE moat patent.
4. **Cost-to-kernel attribution** — Algo: Q_COST_TO_KERNEL join (D.7) with time-share weighting and committed-spend deconvolution (#13 folds in as dependent claims). Distinguish: Kubecost-class pod-level allocation. Value: unique unit of account.
5. **Roofline-plus learned-residual digital twin** — Algo: RFC-004 §4.1. Claims: hybrid analytical/learned simulator where residual model is trained on cross-installation measurement corpus keyed by kernel identity, with data-support-aware uncertainty. Distinguish: pure analytical simulators (AMPed/Calculon-class), pure ML predictors. Value: savings baselines + planning.
6. **SASS-schedule inference under opacity** — Algo: probabilistic decode calibrated against measured counters (OQ-01). Claims: estimating undocumented machine-code scheduling behavior by joint inference over opcode tables and hardware counter observations. Value: NVIDIA-independence of analysis depth.
7. **Counterfactual savings verification** — Algo: RFC-004 §4.3 (frozen-state replay against actual demand + Shapley action attribution + co-signed re-anchoring). Distinguish: trailing-baseline FinOps savings claims. Claims: computing verified savings by simulating a cryptographically-frozen baseline configuration against realized demand. Value: enables savings-share contracts.
8. **MLIR-centric multi-accelerator plugin contract** — Arch: RFC-005/002 backend plugin interface. Claims: uniform kernel-analysis interface over heterogeneous accelerators via dialect-lowering plugins with capability negotiation. Value: cross-vendor scalability.
9. **Compiler-version governance ledger** — Arch: RFC-009 I.8 chain + toolchain approval states + policy gating. Claims: hash-chained approval ledger controlling deployment of compiler toolchains with automated regression evidence attachment. Value: regulated-industry wedge.
10. **Fusion opportunity detector** — RFC-002 pattern 1/7 with graph-prior gain estimation. Dependent on #1/#3 corpus.
11. **Quantization opportunity + accuracy-risk estimator** — pattern 5 + risk model; claims tie recommendation to policy-gated risk score.
12. **Multi-tenant per-request GPU attribution** — RFC-004 §4.7 phase-weight model with published error bounds. Distinguish: naive token-proportional billing.

17. **KES-gated kernel marketplace** — provenance + score-verified listing. Tier-3 (Y4 product).
