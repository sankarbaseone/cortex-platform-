# GAP-007: T-005's "shipped" ci.yaml does not exist; no live location named for genmanifests output
Status: Accepted (founder-resolved)
## Affected RFC/ECD
`05-manifests/build-order.yaml` (T-005 `ci-live` files: `".github/workflows/ci.yaml (shipped)"`) vs. `02-ecd/ECD-010-infrastructure-construction.md` ("Shipped artifacts ... `artifacts/infra/gha/ci.yaml` — complete per-PR workflow ...") vs. the actual specification archive contents; and separately, `02-ecd/ECD-001-repository-construction-plan.md` §2 ("Rule: any new top-level directory requires an ADR + RFC_CONFLICT check. None are anticipated for V1.0") vs. `02-ecd/ECD-006-database-construction.md` / `02-ecd/ECD-002-source-code-construction-plan.md` describing `database-registry.yaml`, `api-registry.yaml`, `metrics-registry.yaml`, etc. as genmanifests-generated without stating where the generated (non-archive) copy lives in the live tree.
## The ambiguity
1. `build-order.yaml` marks T-005's `.github/workflows/ci.yaml` as "(shipped)", and ECD-010 claims `artifacts/infra/gha/ci.yaml` is a complete shipped artifact. No `infra/` directory (or any `gha`/`ci.yaml` file) exists anywhere under `docs/specification/v1.0/07-artifacts/` — confirmed by a full recursive listing. Every other artifact ECD-010 claims as "shipped" (helm chart, Dockerfile, docker-compose, clickhouse terraform module) is actually present in `07-artifacts/`; only the CI workflow is absent.
2. Separately, ECD-006/ECD-002 describe several registries (`database-registry.yaml`, `api-registry.yaml`, `metrics-registry.yaml`, ...) as generated/kept-in-sync by `tools/genmanifests`, distinct from the frozen, do-not-modify copies already sitting in the archive (`docs/specification/v1.0/05-manifests/*.yaml`, confirmed read-only per GAP-004). No frozen document states where genmanifests' live-repo output should be written, and ECD-001 §2's normative tree has no top-level `registries/` (or similar) directory — and explicitly forbids adding one without an ADR.
## Reason
(1) is the same class of gap as GAP-002 (an ECD claims an artifact is "shipped" that was never actually included in the frozen corpus). (2) is an omission: the ECDs describe *what* genmanifests generates and *that* it must stay in sync, but not *where* the generated file lives once produced, because the archive path was reused as a stand-in ("this is the seed") without a corresponding live-tree path ever being specified.
## Resolution
Founder-approved, narrowest-reading fallback for both:
1. `.github/workflows/ci.yaml` is authored directly from RFC-011 §J.3 (pipeline stages) and §Q (quality gate table) — the authoritative Level-1 content ECD-010's missing artifact would have implemented — plus ECD-010's one-line job description (mise pin, gen-drift check, buf breaking, gitleaks/semgrep/osv, privacy-egress check, build matrix, kind e2e smoke) for job naming/shape. Stages whose implementation is scheduled to a later build-order task (image build matrix, kind e2e smoke, coverage gate, benchmark diff, SBOM+sign, schemathesis fuzz, privacy-egress check) are present as named CI jobs that no-op/skip with an explicit message, exactly mirroring the graceful-skip convention already established in `tools/mk/{go,dev,release}.mk` by T-001-T-004 ("empty green pipeline" per ECD-015).
2. genmanifests' generated registries are written under `tools/genmanifests/registries/` — nested inside the already-sanctioned `tools/genmanifests/` path from ECD-001 §2, not a new top-level directory, so no ADR is triggered. `docs/specification/v1.0/05-manifests/*.yaml` remains untouched (GAP-004).
## Narrowest-reading fallback
Applied: T-005 ships `.github/workflows/ci.yaml` authored from RFC-011 text (not copied from a nonexistent artifact), `tools/semgrep/` custom rules, and `tools/genmanifests/` with output nested at `tools/genmanifests/registries/`. The DoD's drift-check proof uses tool-owned test fixtures (not a live scan of T-004's in-flight migrations, which T-005 does not depend on) so the check is deterministic regardless of sibling-task progress.
