# GAP-003: T-001 scope conflict — build-order.yaml vs ITP-03 §2
Status: Accepted (founder-resolved)
## Affected RFC/ECD
`05-manifests/build-order.yaml` (T-001 entry) vs `04-itp/NYDUX-ITP-03-claude-code-operating-manual.md` §2 ("Repository initialization — Day 1 = task T-001").
## The ambiguity
build-order.yaml scopes T-001 narrowly: repository skeleton + `.tool-versions`, `Makefile`, `buf.yaml`, `.golangci.yml`, `CLAUDE.md`, `README.md`; verify `make bootstrap && make lint`. It explicitly assigns protos to T-002, migrations to T-004, CI to T-005, runbooks/dashboards to T-601. ITP-03 §2 describes T-001 as also copying ALL shipped artifacts (protos, migrations, chart, CI, registries, runbooks, dashboards, Rego) in one shot on Day 1 — directly contradicting build-order.yaml's task split.
## Reason
Two frozen documents disagree on one task's scope.
## Resolution
Founder-approved: build-order.yaml (a machine-readable manifest, rank 1 in the stated authority order) is authoritative over ITP-03 (rank 4). T-001 is limited to repository foundation only — skeleton, toolchain/build config, CLAUDE.md, README.md. Protobuf, SQL migrations, CI workflows, Helm, Terraform, dashboards, runbooks, OPA policies, and registries are deferred to their respective build-order tasks and were not copied or implemented in T-001.
## Narrowest-reading fallback
Applied: T-001 created only the files/dirs build-order.yaml's T-001 entry names, plus the ECD-001 top-level directory skeleton needed to support `make bootstrap && make lint`.
